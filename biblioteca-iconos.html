Tutorial: Cómo Usar Iconos

Aprende paso a paso a implementar iconos en tus proyectos

1

Importar Font Awesome en tu HTML

Agrega esta línea en la sección <head> de tu documento HTML:

HTML 
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css">
Nota: Esta línea carga la biblioteca completa de iconos desde un CDN (Content Delivery Network)
2

Usar la Sintaxis Correcta

Los iconos se insertan usando la etiqueta <i> o <span> con clases específicas:

HTML 
<i class="fas fa-home"></i>
<i class="far fa-heart"></i>
<i class="fab fa-github"></i>
3

Estilizar con CSS

Los iconos se comportan como texto, por lo que puedes cambiar su tamaño, color y más:

CSS 
.mi-icono {
    font-size: 2rem;      /* Tamaño */
    color: #667eea;       /* Color */
    margin: 10px;         /* Espaciado */
}

Semántica y Mejores Prácticas

Aprende a usar iconos de manera semánticamente correcta

Cuándo Usar Iconos

  • Para reforzar visualmente el significado de un texto
  • En botones de acción para indicar su función
  • En navegación para mejorar la orientación
  • Para mostrar estado o notificaciones
  • NO uses iconos sin texto descriptivo
  • NO uses iconos para transmitir información crítica sola

Jerarquía Visual

Iconos principales (acciones importantes)
Iconos secundarios (información complementaria)
Iconos decorativos (elementos visuales)

Consistencia de Color

Acciones primarias
Estados positivos/éxito
Estados negativos/error
Advertencias/atención

Ejemplos de Uso Semántico

✅ Correcto: Icono con texto descriptivo

<button class="btn">
    <i class="fas fa-shopping-cart" aria-hidden="true"></i>
    Agregar al Carrito
</button>

✅ Correcto: Icono solo con aria-label

<button class="btn-icon" aria-label="Cerrar ventana">
    <i class="fas fa-times" aria-hidden="true"></i>
</button>

❌ Incorrecto: Icono sin contexto

<!-- Mal: No hay texto ni aria-label -->
<button>
    <i class="fas fa-heart"></i>
</button>

Accesibilidad en Iconos

Haz tus iconos accesibles para todos los usuarios

Para Usuarios con Discapacidad Visual

Los lectores de pantalla necesitan contexto para describir los iconos

  • Usa aria-label cuando el icono está solo
  • Usa aria-hidden="true" cuando hay texto descriptivo
  • Nunca dependas solo del icono para transmitir información

Navegación por Teclado

Asegura que los iconos interactivos sean accesibles por teclado

  • Usa elementos <button> o <a> para iconos clickeables
  • Agrega tabindex="0" si usas otros elementos
  • Implementa estados :focus visibles

Contraste de Color

Mantén suficiente contraste entre iconos y fondo

  • Mínimo 4.5:1 para texto normal (WCAG AA)
  • Mínimo 3:1 para iconos grandes (18pt+)
  • No uses solo color para transmitir información
